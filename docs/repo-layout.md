# Repo layout

This repo is the private canonical `bowline` workspace. Public repos are exports
from this repo, not separately edited products.

## Top-level shape

```text
apps/         deployable TypeScript surfaces
packages/     TypeScript deep modules shared by apps
crates/       Rust CLI, daemon, local engine, and storage substrate
docs/         product, architecture, research, and repo docs
docs/implementation/
              implementation planning, phase gates, and smoke evidence
research/     source material, Oracle output, and subagent notes
reports/      generated analysis reports
scripts/      architecture and quality gates
transcripts/  preserved conversation exports
```

The root keeps only operational entrypoints such as `README.md`, `AGENTS.md`,
`CONTEXT.md`, workspace manifests, and tool configuration.

## Cloud app shape

The v1 cloud product shape is TanStack Start on Cloudflare Workers,
WorkOS/AuthKit for account identity, hosted Convex for the control plane, and
Cloudflare R2 for immutable encrypted packs and manifests.

Use these boundaries for the TypeScript side:

- `apps/web/` is the TanStack Start web and dashboard surface.
- `apps/api-worker/` is a temporary thin Cloudflare Worker scaffold for
  edge/status experiments. It is not the sync backend, byte proxy, or control
  plane.
- `packages/control-plane/` owns product-shaped control-plane contracts and
  helpers. It must not become a raw table-access layer leaked across the repo.
- `infra/cloudflare/` owns Cloudflare deployment configuration when needed.
- `infra/object-storage/` owns R2 bucket and object-storage configuration when
  needed.
- `infra/schema/` is the home for provider-specific schema notes only when a
  real external service needs them.

## Boundary rules

- Apps are thin deployables. Product logic lives in packages or crates.
- TypeScript packages export public entrypoints from `src/index.ts`.
- Code outside a module must not import from `internal/`.
- Rust crates keep internals private by default.
- Generated output must stay out of tracked source paths.
- The Rust daemon talks to hosted Convex through a product-shaped
  `ControlPlaneClient` boundary and uploads encrypted packs and manifests
  directly to R2.
- Approved devices are workspace-wide trusted devices. Agent leases scope agent
  behavior.
- The private repo is canonical. Public repos are generated by the allowlist in
  `public-export.json`, not edited as separate source trees.
- `AGENTS.md`, private plans, transcripts, raw research, generated reports, env
  files, production deployment wiring, and billing/admin/support internals stay
  out of public exports.

## Verification

Use the root command:

```bash
pnpm verify
```

For the generated public source repo:

```bash
pnpm verify:public
```

Maintainers can update a public working tree from the private repo with:

```bash
pnpm export:public --target <public-repo>
pnpm check:public-export --root <public-repo>
pnpm deploy:public --target <public-repo>
```

`deploy:public` only commits generated public source. `pnpm ship` is the private
top-level runtime deploy for public source, Convex, Cloudflare, and production
URL smokes. `pnpm release --version <semver>` is the package/binary release
entrypoint for downloadable assets served by `install.bowline.sh`.

For Rust-only work:

```bash
pnpm rust:verify
```
